28 20 KEY DECISIONS
20个核心设计决策 （1~10）
01
State 全量替换模式
queryLoop 每个站点 state = {...新对象}，不逐字段赋值，隔离性强，状态变迁可追踪
02
in-place mutation 的例外
lastMsg.usage = usage，因 transcript write queue 持有引用，100ms flush 间隔替换会丢数据
03
自实现 O(n) 流式解析
SDK partialParse() 是 O(n²)，Claude Code 增量拼接，content_block_stop 时一次性 parse
04
Prompt Cache Eligible Latch
session 开头锁定，不再更新。mid-session 超额后 TTL 回退 ephemeral，改字段会触发 ~20K token bust
05
三级压缩成本梯度

33 OPENCLAW ENGINEERING
OpenClaw 最值得借鉴的 8个工程设计
01
eventGate 防超时污染
turn 超时后 eventGate.open=false，stream 继续流但事件被过滤，防止延迟事件污染下一个 turn。比 abort+catch 更优雅
02
Auth Round-Robin 反直觉设计
lastGood 故意不参与排序（否则是"总用最近成功的"，不是真正的 round-robin）。OAuth > token > api_key 自然降级
03
孤儿恢复 + 动态 import
scheduleOrphanRecovery 通过 void import('./subagent-orphan-recovery.js') 延迟加载，不增加 gateway 启动内存，失败静默忽略（best-effort）
04
cron startup 防雪崩
重启后最多补跑 5 个错过的 job，每个错开 5s，25s 内完成所有补跑，不会在重启时产生 thundering herd
05
canAutoSelectLocal 负向约束
auto embedding 主动排除 Ollama（不能假设用户有）和 hf: 前缀路径（不能假设有下载权限），只允许已存在本地文件参与
06
DraftStreamLoop 覆盖设计
流式 typing 不用队列，用覆盖（pendingText=text），配合 throttleMs，永远发最新内容而非历史积压，与 Claude Code pendingToolUseSummary 同一哲学
07
28强类型 HookRunner
全生命周期 28 个 TypeScript 函数（Agent/Message/Tool/Subagent/Gateway/Install），返回修改后对象表达「变换」语义，远优于 shell exit-code 协议
08
安全审计动态加载
深度检查、渠道检查模块都是运行时动态 import，不影响 gateway 启动时间和内存；SecurityAuditSeverity 三级集成进 /status 命令
