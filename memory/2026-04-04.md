### QueryEngine.ts 深度分析要点

**架构定位：**
- 对话生命周期所有者：一个实例 = 一个完整对话
- 位于 query.ts 之上、main.tsx 之下
- processUserInput() → QueryEngine.submitMessage() → query() → claude.ts → Anthropic API

**QueryEngineConfig 23 个关键字段：**
- tools, commands, mcpClients, agents（工具/命令/MCP/agent 集合）
- canUseTool（权限检查回调）
- customSystemPrompt（替换默认）/ appendSystemPrompt（追加尾部）
- userSpecifiedModel / fallbackModel / thinkingConfig
- maxTurns / maxBudgetUsd / taskBudget
- jsonSchema（结构化输出）
- snipReplay（HISTORY_SNIP 注入点，通过 config 传入而非直接调用）

**11个执行阶段：**
1. Turn 开始清理（discoveredSkillNames.clear(), setCwd）
2. 权限追踪包装（透明代理，记录 permissionDenials）
3. System Prompt 构建（前缀 cache 优化，顺序关键）
4. processUserInputContext 两次构建（第一次可写，第二次 noop）
5. Transcript 提前写入（API 调用前持久化，容灾设计）
6. shouldQuery 分流（本地 slash 命令 vs API 调用）
7. `for await` 核心消息流循环
8. compact_boundary 后 splice（防内存泄漏）
9. snipReplay 注入（feature-gated 字符串隔离）
10. errorLogWatermark 引用水印（环形缓冲区安全）
11. 5种终止结果类型

**关键设计决策：**
- assistant 消息 fire-and-forget（配合 claude.ts 原地 mutation + lazy JSON，防生成器阻塞）
- FileStateCache Clone-on-call（ask() 克隆，finally 写回，隔离性+一致性）
- isPartialView 标志（CLAUDE.md 注入剥离后，Edit/Write 需先 Read）
- snipReplay 通过 config 注入而非直接调用（excluded-strings 隔离）
- 引用水印 vs length-index（环形缓冲区 shift 时 length-index 会滑动）

**utils/queryContext.ts：**
- fetchSystemPromptParts()：并行获取 defaultSystemPrompt + userContext + systemContext
- customSystemPrompt 存在时跳过 getSystemContext()
- 工具函数供 QueryEngine + cli/print.ts 共用

**utils/fileStateCache.ts：**
- LRUCache 包装（100条/25MB）
- normalize() 路径标准化（防止 /foo/../bar 路径 cache miss）
- FileState: content（原始磁盘内容）/ timestamp / offset / limit / isPartialView

### 发现的有趣内容
- **"Tengu"（天狗）**：内部代号，analytics event 名称前缀（tengu_api_success, tengu_startup_telemetry 等）
- **Buddy/Companion 系统**：Tamagotchi 风格宠物（鸭子等），mulberry32 PRNG 生成外观特征
- **Dream 系统**：会话结束后记忆整合（独立子进程 forked agent，分享父进程 prompt cache）
- **KAIROS**：多智能体编排，in_process_teammate 任务类型
- **Undercover Mode**：防止泄露 Claude Code 正在运行（process 名称伪装）
- **Coordinator Mode**：feature-gated，注入额外 userContext
- **Claude Opus 4.6**：在源码中可见，尚未公开发布
- **SSH Remote (SSH_REMOTE feature)**：远程 SSH 连接支持，v1 不支持 headless 模式
- **Teleport**：远程会话续传功能
